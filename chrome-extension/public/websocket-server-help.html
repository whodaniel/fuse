WebSocket Server Setup - The New Fuse

The WebSocket server is required for The New Fuse Chrome extension to communicate with VS Code.

We tried to open VS Code to start the server, but it seems the VS Code integration might not be installed or properly set up.

How to Start the WebSocket Server

1Open VS Code

Make sure VS Code is installed and open.

2Run the WebSocket Server Task

In VS Code, press Ctrl+Shift+P (Windows/Linux) or Cmd+Shift+P (Mac) to open the command palette.

Type "Tasks: Run Task" and select it.

Then choose "Start WebSocket Server" from the task list.

Tasks: Run Task > Start WebSocket Server

3Verify the Server is Running

You should see a message in the VS Code terminal indicating that the WebSocket server has started.

The server typically runs on port 8080 by default.

Note: If the VS Code extension for The New Fuse is not installed, you'll need to install it first:

  1. Open VS Code
  2. Go to Extensions (Ctrl+Shift+X or Cmd+Shift+X)
  3. Search for "The New Fuse"
  4. Install the extension

Manually Start the Server

If you prefer to start the server manually, you can run the following command in your terminal:

cd /path/to/your/project && node test-websocket-server-3711.cjs

Make sure to replace the path with your actual project directory.

After starting the server, go back to the Chrome extension and click "Connect to WebSocket Server" to establish the connection.

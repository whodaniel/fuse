Consolidated Tech Stack

I. Architecture & Monorepo Management

  • Package Manager: Yarn (specifically Yarn 4.9.1 with Berry mentioned in TECH-STACK.md and root package.json)
  • Workspace Structure: Multi-package monorepo (apps/ and packages/ structure)
  • Build System: Yarn workspaces

II. Frontend (apps/frontend)

  • Core Framework: React 18 (Functional Components, Hooks)
  • Language: TypeScript
  • Build Tool: Vite
  • UI Component Libraries:
    • Primary: Chakra UI
    • Headless: Radix UI
    • Secondary/Legacy: Material-UI (MUI)
  • Icons: Heroicons, Phosphor Icons, Lucide React
  • Styling:
    • Tailwind CSS (utility-first)
    • Emotion (CSS-in-JS)
  • Animations: Framer Motion
  • State Management:
    • Global: Redux Toolkit
    • Local/Component: Zustand
  • Server State/Caching: TanStack Query (React Query)
  • Forms: React Hook Form
  • Routing: React Router DOM
  • HTTP Client: Axios
  • Real-time Communication: WebSockets (ws library, Socket.IO client-side if backend uses it)
  • In-browser Code Editor: Monaco Editor (@monaco-editor/react)
  • Data Visualization:
    • D3.js
    • Recharts
    • ReactFlow (for flow/diagrams)
  • Internationalization (i18n): i18next, react-i18next

III. Backend (apps/api, apps/backend - often NestJS based)

  • Core Framework: NestJS
  • Language: TypeScript
  • Runtime: Node.js (ES Modules support)
  • Database & ORM:
    • Primary Database: PostgreSQL
    • Secondary/Other: MongoDB (mentioned in ARCHITECTURE.md)
    • ORM/Query Builders: Prisma (modern toolkit), TypeORM (decorator-based)
  • Caching: Redis (ioredis client)
  • Authentication & Authorization:
    • JWT (JSON Web Tokens)
    • Passport.js with strategies (JWT, Google OAuth)
    • Password Hashing: bcrypt
  • Real-time Communication:
    • WebSockets (Socket.IO via @nestjs/websockets, @nestjs/platform-socket.io)
    • Redis Pub/Sub (for distributed messaging, mentioned in ARCHITECTURE.md and AGENT-COMMUNICATION-GUIDE.md)
    • Redis Streams (for reliable, persistent messaging as per AGENT-COMMUNICATION-GUIDE.md)
  • API Documentation: OpenAPI (Swagger via @nestjs/swagger)
  • Validation: class-validator, class-transformer, Zod
  • Configuration: dotenv

IV. VS Code Extension (src/vscode-extension)

  • Language: TypeScript

IX. Protocols & Communication Systems

  • Model Context Protocol (MCP): A core protocol for AI agent communication, tool usage, and context sharing. The New Fuse acts as both an MCP server and client.
    • Uses WebSocket and JSON-RPC.
    • Features include agent registration, capability declaration/assessment, tool discovery, and execution.
    • MCPBrokerService and DirectorAgentService are key backend components.
    • Refer to MCP-GUIDE.md and src/mcp/README.md.
  • Agent-to-Agent (A2A) Communication:
    • Relies heavily on Redis Streams for reliable, persistent, asynchronous messaging.
    • Uses standardized JSON message formats.
    • Supports task distribution, event broadcasting, and direct messaging.
    • Refer to AGENT-COMMUNICATION-GUIDE.md.
  • Redis: Used extensively for:
    • Message Queuing (Streams)
    • Caching (Strings with expiration)
    • Pub/Sub (though Streams are preferred for A2A)
    • Shared State Management (Hashes for task registries, LLM memory)
    • Distributed Locking (SET NX PX)
    • Vector Database capabilities (RediSearch/Redis Stack for indexing code and semantic search)
